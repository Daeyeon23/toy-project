Notes

앱 진입 시 빵 계열을 고르는 첫 화면. 식빵만 진단 가능하고 나머지 계열은 방향만 보여준다.

  • 트리거: 앱 최초 진입.
  • 인터랙션: "식빵" 카드 탭 → 진단 화면으로 이동 / "준비 중" 카드는 비활성이라 탭해도 이동하지 않음.
  • 전환: 식빵 선택 → 진단 화면.
  • 불변: 헤더에 "기본 이스트 식빵 기준" 범위 고지 항상 노출.

실패한 식빵의 증상을 골라 주세요. 여러 개일수록 원인이 좁혀집니다.

검색은 보조 수단입니다. 아래에서 바로 선택해도 됩니다.

Notes

선택한 종류(식빵)의 실패 증상을 다중 선택해 진단을 시작하는 화면. 상단 검색은 목록을 좁히는 보조 수단.

  • 트리거: 빵 선택 화면에서 "식빵"을 고른 뒤 진입. 증상 0개 상태로 시작 → 진단하기 비활성.
  • 인터랙션: 증상 탭 → 체크 토글 / 검색어 입력 → 목록 필터(증상 검색 화면) / 증상 1개 이상 → 진단하기 활성.
  • 전환: "진단하기" → 상위 원인이 근접하면 판별 질문, 명확하면 결과 화면.
  • 불변: 범위 고지 배지 항상 노출.

Notes

진단 화면의 검색 필터가 적용된 상태. 일상어를 미리 정의된 동의어 사전에 매칭해 목록을 좁힌다(위), 매칭이 없으면 빈 결과 안내(아래).

  • 트리거: 진단 화면 검색란에 텍스트 입력.
  • 인터랙션: 좁혀진 증상 탭 → 선택됨(선택은 검색어 지워도 유지) / X 또는 검색어 삭제 → 전체 목록 복원 / no-match → 안내 노출.
  • 전환: 이 화면은 진단 화면의 필터 상태 — 선택 후 진단 흐름은 진단 화면과 동일.
  • 불변: 범위 고지 배지 항상 노출.

1차 발효 때 반죽이 부풀긴 했나요?

Notes

상위 원인이 근접(동점)할 때, 결과를 바로 보여주지 않고 둘을 가르는 질문을 던지는 중간 화면.

  • 트리거: 진단 상위 두 원인 점수가 근접할 때 (예: 이스트 문제 ↔ 발효 부족).
  • 인터랙션: "예"/"아니오" → 해당 방향으로 순위 확정 / "건너뛰기" → 근접했던 두 원인을 함께 담아 결과로.
  • 전환: 답변·건너뛰기 → 결과 화면. 추가 근접 쌍이 있으면 질문 2로 이어지고, 최대 2개 후 결과로 이동.
  • 불변: 범위 고지 배지 항상 노출.

단정적 진단이 아니라 일반적 제빵 상식 기반의 가능성 있는 원인입니다.
1 발효 부족 가능성 높음
왜 이 증상이 나오는지

이스트가 가스를 충분히 만들기 전에 구워, 부피가 안 나고 속이 떡지며 팽창 압력이 약한 표면을 터뜨립니다.

다음에 이렇게

따뜻한 곳에서 반죽이 약 2배가 될 때까지(손가락 자국이 천천히 돌아올 때까지) 1차 발효를 늘려 보세요.

2 이스트 문제 가능성 중간
왜 이 증상이 나오는지

이스트가 죽었거나 뜨거운 물에 사멸하면 발효 자체가 시작되지 않아 부풀지 않습니다.

다음에 이렇게

유통기한을 확인하고, 물 온도를 40℃ 이하로 맞춘 뒤 예비 발효로 활성을 확인하세요.

3 저온 환경 가능성 중간
왜 이 증상이 나오는지

반죽·실내 온도가 낮으면 발효 속도가 크게 느려져 같은 시간에 덜 부풉니다.

다음에 이렇게

오븐 발효 기능이나 따뜻한 물 옆 등 26~28℃ 환경에서 발효시켜 보세요.

Notes

가능성 높은 순으로 정렬된 원인 카드 2~3개를 보여주는 결과 화면. 각 카드는 순위 · 원인명 · 강도 라벨 · "왜 이 증상이 나오는지" · "다음에 이렇게"로 구성.

  • 트리거: 진단하기 직후(명확한 경우) 또는 판별 질문 응답·건너뛰기 후 진입.
  • 인터랙션: 카드를 위→아래로 순위대로 읽음. 1순위는 테두리·번호로 강조. 강도는 "가능성 높음/중간" 라벨로만 표현.
  • 전환: "증상 바꿔 다시 진단" → 진단 화면으로 이동, 이전 결과는 사라짐.
  • 불변: 면책 고지 항상 노출 / 범위 고지 항상 노출 / 원시 점수·퍼센트는 노출하지 않음(순위·라벨만).
  • data-scenario: 건너뛰기 시(scenario-4)에는 근접했던 두 원인 카드가 함께 표시됨.
