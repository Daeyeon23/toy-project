# 브레드 닥터 (Bread Doctor) — Spec

## 개요
직접 구운 빵이 실패한 홈베이커가, **만들던 빵 종류를 고르고 → 관찰한 증상을 선택**하면
**가능성 높은 원인을 순위로** 받고 "왜 그런지 + 다음에 고칠 것"을 확인하는 진단 도구다.
상위 원인이 서로 비슷해 가려지지 않을 때는 **판별 질문**으로 좁힌다. 모든 진단은 브라우저
안에서 끝난다(네트워크 없음).

> **v2 확장:** 초기 MVP는 기본 이스트 식빵 1종만 제공했다. 식빵으로 판별력이
> 증명되었으므로(MBT#1 CONFIRMED), 이제 **25종의 빵**으로 확장한다 — 각 빵은 종이 위
> 증상×원인 매트릭스 검증(`artifacts/bread-doctor/matrices/*.md`, 전부 PEAKED 판정)을
> 거쳐 자체 증상·원인·판별 질문·동의어 세트를 갖는다. 진단은 **선택한 빵 종류 기준**으로
> 이뤄지며, 빵마다 증상 체크리스트·결과가 달라진다.

> **v3 확장 (본 스펙):** v2는 빵 25종까지 다뤘다. 이제 **제과류를 파일럿 8종** 추가한다 —
> 실패 메커니즘이 서로 다른 8개 카테고리(유화 배터 / 멜팅버터·크리밍 / 러빙 / 머랭 거품 /
> 전란 거품 스펀지 / 분리교반 / 화학팽창 머핀 / 스팀 팽창)에서 대표 1종씩(휘낭시에·브라우니·
> 타르트·마카롱·붓세·시폰케이크·머핀(퀵브레드)·슈)을 고른다(`artifacts/bread-doctor/idea-pastry.md`
> 참고). "빵"에 한정됐던 화면·문구는 **"베이커리 품목"**으로 일반화하고, 진단 구조(증상
> 체크리스트 → 순위 결과 → 근접 시 판별 질문)는 v2와 동일하게 재사용한다. 나머지 8종
> (마들렌·티그레·버터바 등)과 카테고리 세부 확장은 이번 스펙 밖이며, 파일럿으로 카테고리별
> 진단축이 실제로 다르다는 게 검증된 뒤 별도로 다룬다.

## 범위

### 포함
- **베이커리 품목 선택 화면** — **33개 품목(빵 25종 + 제과류 파일럿 8종)이 16개 카테고리로
  그룹핑**되어 표시되고, 모두 선택 가능
  (빵 8개 카테고리: 식빵·린도우 / 사워도우·호밀 / 라미네이션 / enriched 단맛반죽 / 화학팽창 /
  발효 플랫브레드 / 무발효 플랫브레드 / 보일링 — 제과류 8개 카테고리: 유화 배터 / 멜팅버터·
  크리밍 / 러빙 / 머랭 거품 / 전란 거품 스펀지 / 분리교반 / 화학팽창 머핀 / 스팀 팽창)
- **베이커리 품목 검색·필터** — 품목 이름 일부를 입력하면 매칭되는 품목으로 그리드가 좁혀짐
- **선택한 품목 기준**의 증상 기반 진단 — 품목마다 자체 증상·원인·판별 질문·동의어 세트를 사용
- 증상 체크리스트(다중 선택)에서 증상 선택 — **목록은 선택한 품목의 증상 세트**
- **증상 검색·동의어 필터** — 일상어("떡져요")를 입력하면 해당 증상으로 목록이 좁혀짐 (**선택한 품목의 동의어 사전** 기준)
- 상위 2~3개 원인을 순위로 제시, 각 원인마다 "원인 / 왜 이 증상이 나오는지 / 다음에 이렇게" 표시
- 상위 원인이 근접할 때만 판별 질문(1개)을 띄워 순위를 좁힘
- 증상을 바꿔 다시 진단 / **품목을 바꿔 다시 진단**
- 결과에 "단정적 진단이 아니라 일반적 베이킹 상식 기반" + **"○○(품목 이름) 기준"** 안내 노출
- **이름이 비슷한 기존 항목과의 구분** — 빵 "잉글리시 머핀"(이스트 반죽)과 신규 제과류
  "머핀(퀵브레드)"(화학팽창)처럼 이름이 겹치는 품목은 서로 다른 카테고리·라벨로 구분 표시

### 제외
- **자유 텍스트 해석(LLM/키워드 파싱)** — 결정론(테스트 가능성)을 무너뜨림. 검색·동의어 필터는 *미리 정의된 동의어 사전*으로만 동작하며, 최종 선택은 항상 체크리스트 항목이다.
- **판별 질문 체이닝(2개 이상 연속)** — 한 번의 진단에서 판별 질문은 **최상위 근접 쌍 1개에 대해 최대 1개만** 뜬다. 빵마다 여러 근접 쌍(따라서 여러 판별 질문)이 정의되지만, 이는 서로 다른 증상 조합에 대응하는 것이며 한 진단에서 연속으로 묻지 않는다. (`MAX_DISCRIMINATOR_QUESTIONS` 캡은 그대로 휴면 상태로 남는다.)
- **사진 업로드/AI 판독** — 범위 과대. 체크리스트로 가치 증명.
- **계정·로그인·진단 기록 저장** — 데모/포트폴리오 핵심 아님.
- **원시 점수/퍼센트 노출** — 사용자에겐 순위와 정성적 강도만. 스코어는 내부 계산.
- **인용 출처의 UI 노출** — 매트릭스 문서(`matrices/*.md`)의 출처 각주는 KB 저술 근거이며, 앱 화면에는 노출하지 않는다(불변 규칙의 면책 고지로 갈음).

## 시나리오

### 0. 베이커리 품목 선택 (33종, 카테고리 그룹)
- **Given** — 앱 진입 시 베이커리 품목 선택 화면에 33개 품목(빵 25종 + 제과류 8종)이 16개
  카테고리로 묶여 표시됨
- **When** — 아무 품목 카드(예: "사워도우", 또는 "마카롱")를 선택함
- **Then** — **그 품목의** 증상 체크리스트 화면으로 이동함

성공 기준:
- [ ] 선택 화면에 33개 품목 카드가 표시되고, 각 카드는 카테고리 그룹(예: "사워도우·호밀", "머랭 거품") 아래에 묶여 있다
- [ ] 모든 품목 카드가 선택 가능하다 ("준비 중"/비활성 카드가 없다)
- [ ] 임의의 품목 카드 선택 → 증상 체크리스트 화면이 표시된다
- [ ] 진입 시 어떤 품목도 선택되지 않은 상태이며, 품목을 고르기 전에는 증상 화면으로 갈 수 없다

### 0-A. 선택한 품목마다 증상·원인 세트가 다르다
- **Given** — 베이커리 품목 선택 화면
- **When** — "크루아상"을 선택함
- **Then** — 크루아상 고유 증상(예: "층이 안 보여요", "버터가 새어 나옴")이 포함된 체크리스트가 표시됨. 식빵을 골랐을 때와 증상 목록이 다름

성공 기준:
- [ ] "크루아상" 선택 → 체크리스트에 라미네이션 관련 증상(`lam-*`)이 표시된다
- [ ] "식빵" 선택 → 체크리스트에 식빵 증상 세트가 표시되고, 크루아상 전용 증상은 보이지 않는다
- [ ] 선택한 품목으로 진단 → 그 품목의 원인 세트에서만 원인 카드가 나온다 (다른 품목의 원인이 섞이지 않는다)

### 0-A'. 제과류를 선택해도 동일한 구조를 따른다
- **Given** — 베이커리 품목 선택 화면
- **When** — 제과류 카테고리의 "마카롱"을 선택함
- **Then** — 마카롱 고유의 증상 체크리스트(머랭 거품 카테고리에 해당하는 증상 세트)가 표시됨.
  다른 제과류(예: "휘낭시에")나 빵을 골랐을 때와 증상 목록이 다름

성공 기준:
- [ ] "마카롱" 선택 → 체크리스트에 마카롱 전용 증상 세트가 표시된다
- [ ] "휘낭시에" 선택 → 체크리스트에 마카롱과 다른, 휘낭시에 전용 증상 세트가 표시된다
- [ ] 마카롱으로 진단 → 마카롱의 원인 세트에서만 원인 카드가 나온다 (빵이나 다른 제과류의 원인이 섞이지 않는다)

> 파일럿 8종 각각의 구체적 증상·원인 문구는 매트릭스 문서(`artifacts/bread-doctor/matrices/*.md`
> 신규분) 작성 후 확정된다 — 의존성 참고.

### 0-B. 품목 이름으로 검색해서 찾기
- **Given** — 베이커리 품목 선택 화면에 검색 입력란이 표시됨
- **When** — 검색란에 "치아"를 입력함
- **Then** — 그리드가 "치아바타"로 좁혀짐

성공 기준:
- [ ] "치아" 입력 → "치아바타" 카드만 남고 무관한 품목은 보이지 않는다
- [ ] 매칭되는 품목이 없는 검색어 입력 → "일치하는 품목 없음" 안내가 표시된다
- [ ] 검색어를 지우면 33종 전체 그리드가 복원된다

### 0-C. 품목을 바꿔 다시 진단
- **Given** — 어떤 품목(예: "식빵")으로 진단 결과가 표시된 상태, **또는** 그 품목의 증상 선택 화면이 표시된 상태(아직 진단하기 전)
- **When** — "다시 고르기"를 눌러 다른 품목(예: "베이글", 또는 빵→제과류로 "마카롱")을 선택함
- **Then** — 증상 선택·이전 결과가 **초기화**되고, 새로 고른 품목의 증상 체크리스트가 표시됨

성공 기준:
- [ ] 결과 화면에서 다시 고르기 → 품목 선택 화면으로 돌아간다
- [ ] **증상 선택 화면(진단하기 전)에서도** 다시 고르기 → 품목 선택 화면으로 돌아간다
- [ ] 다른 품목 선택 → 이전 품목의 증상 선택과 결과가 남아 있지 않다 (초기화)
- [ ] 빵("식빵")에서 제과류("마카롱")로 바꿔도 동일하게 초기화되고, 마카롱의 증상 세트로 진단이 진행된다 (카테고리 경계를 넘어도 동작 동일)

### 0-E. 이름이 비슷한 기존 항목과 새 항목을 구분해서 찾는다
- **Given** — 베이커리 품목 선택 화면 (빵 "잉글리시 머핀"과 제과류 "머핀(퀵브레드)"가 모두 존재)
- **When** — 검색란에 "머핀"을 입력함
- **Then** — "잉글리시 머핀"과 "머핀(퀵브레드)" 두 카드가 모두 나타나며, 각각 다른 카테고리
  그룹(발효 플랫브레드 / 화학팽창 머핀) 아래에 놓여 구분됨

성공 기준:
- [ ] "머핀" 검색 → "잉글리시 머핀"과 "머핀(퀵브레드)" 카드가 모두 표시된다
- [ ] 두 카드는 서로 다른 카테고리 그룹 아래에 위치한다
- [ ] "잉글리시 머핀" 선택 → 기존 이스트 반죽 증상 세트가 표시된다 (v2와 동일, 회귀 없음)
- [ ] "머핀(퀵브레드)" 선택 → 신규 화학팽창 머핀 증상 세트가 표시된다

---

> **아래 시나리오 1~7은 "식빵(white-loaf)" 기준의 대표 예시다.** 초기 MVP에서 확정·테스트된
> 계약이며, v2/v3에서도 식빵 경로의 회귀 가드로 그대로 유지된다. 다른 32종(빵 24종 + 제과류
> 파일럿 8종)도 동일한 구조(증상 조합 → 순위 결과, 근접 시 판별 질문, 동의어 검색)를 따르되
> 구체적 증상·원인·질문은 각 매트릭스 문서를 따른다.

### 1. 명확한 진단 (판별 질문 없이 바로 결과) — 식빵 예시
- **Given** — 빵 선택에서 "식빵"을 고른 뒤, 증상 체크리스트가 표시됨
- **When** — "거의 부풀지 않음", "속이 떡짐/질음", "옆구리·표면이 터짐"을 선택하고 "진단하기"를 누름
- **Then** — 판별 질문 없이 결과가 나타나고, 1순위 원인으로 "발효 부족"이 강조되어 표시됨

성공 기준:
- [ ] 위 3개 증상 선택 → "진단하기" → 판별 질문이 나타나지 않고 결과가 표시된다
- [ ] 결과의 1순위 카드 제목이 "발효 부족"이다
- [ ] 결과에 원인 카드가 2~3개 표시된다
- [ ] 각 원인 카드에 원인명, "왜 이 증상이 나오는지" 설명, "다음에 이렇게" 액션 텍스트가 모두 보인다

### 2. 원인-특이적 증상이 순위를 가른다
- **Given** — 진단 화면
- **When** — "부풀다 주저앉음", "기공이 너무 크고 불규칙", "신맛/이상한 냄새"를 선택하고 "진단하기"를 누름
- **Then** — 1순위 원인으로 "과발효"가 표시됨

성공 기준:
- [ ] 위 3개 증상 → 1순위 카드 제목이 "과발효"다
- [ ] "겉은 탔는데 속은 덜 익음" + "겉이 두껍고 딱딱함" 선택 → 1순위 카드 제목이 "오븐 온도가 높음"이다

### 3. 상위 원인이 근접하면 판별 질문이 뜬다
- **Given** — 진단 화면
- **When** — "거의 부풀지 않음" + "속이 떡짐/질음"만 선택하고 "진단하기"를 누름 (이스트 문제 ↔ 발효 부족이 근접)
- **Then** — 결과를 바로 보여주지 않고, 두 원인을 가르는 판별 질문("1차 발효 때 반죽이 부풀긴 했나요?")이 표시됨

성공 기준:
- [ ] 위 2개 증상 → "진단하기" → 판별 질문이 표시되고 최종 순위는 아직 확정되지 않는다
- [ ] 판별 질문에 "예"로 답 → 1순위 카드 제목이 "발효 부족"이다
- [ ] 판별 질문에 "아니오"로 답 → 1순위 카드 제목이 "이스트 문제"다
- [ ] 판별 질문은 최대 2개까지만 이어지고, 그 후에는 결과가 표시된다

### 4. 판별 질문을 건너뛰면 후보를 함께 보여준다
- **Given** — 판별 질문이 표시된 상태 (시나리오 3)
- **When** — "모르겠어요/건너뛰기"를 선택함
- **Then** — 근접했던 두 원인이 순위 결과에 함께 표시됨 (강제로 하나로 좁히지 않음)

성공 기준:
- [ ] 판별 질문에서 "건너뛰기" → 결과에 "이스트 문제"와 "발효 부족" 카드가 모두 표시된다

### 5. 증상 미선택 시 진단 불가
- **Given** — 어떤 증상도 선택되지 않은 초기 화면
- **When** — 화면을 봄
- **Then** — "진단하기" 버튼이 비활성 상태임

성공 기준:
- [ ] 증상 0개 선택 → "진단하기" 버튼이 비활성(클릭 불가)이다
- [ ] 증상을 1개 선택 → "진단하기" 버튼이 활성화된다

### 6. 증상을 바꿔 다시 진단
- **Given** — 시나리오 2의 결과("과발효" 1순위)가 표시된 상태
- **When** — 증상 선택을 "겉은 탔는데 속은 덜 익음" + "겉이 두껍고 딱딱함"으로 바꾸고 다시 "진단하기"를 누름
- **Then** — 이전 결과가 사라지고 1순위가 "오븐 온도가 높음"으로 갱신됨

성공 기준:
- [ ] 결과 표시 후 증상을 바꾸면 이전 결과 카드가 더 이상 보이지 않는다
- [ ] 바뀐 증상으로 다시 진단 → 새 1순위 원인이 표시된다

### 7. 일상어 검색으로 증상 찾기
- **Given** — 증상 체크리스트 화면에 검색 입력란이 표시됨
- **When** — 검색란에 "떡져요"를 입력함
- **Then** — 증상 목록이 "속이 떡짐/질음"으로 좁혀지고, 탭하면 선택된다

성공 기준:
- [ ] "떡져요" 입력 → 목록에 "속이 떡짐/질음"이 나타나고 무관한 증상은 보이지 않는다
- [ ] 필터된 증상을 탭 → 선택 상태가 되고, 검색어를 지우면 전체 목록이 (선택 유지된 채) 복원된다
- [ ] 동의어 사전에 없는 검색어("우주선") 입력 → "일치하는 증상 없음" 안내가 표시된다
- [ ] 검색어 없이도 전체 체크리스트에서 직접 선택할 수 있다 (검색은 보조 수단)

## 불변 규칙

- **면책 고지**: 모든 진단 결과 화면에 "단정적 진단이 아니라 일반적 베이킹 상식 기반의 가능성 있는 원인"이라는 안내가 항상 표시된다. (v3에서 "제빵" → "베이킹"으로 일반화 — 제과류에는 "제빵(bread-baking)"이 부정확하기 때문)
- **범위 고지**: 진단·결과 화면에 **현재 선택한 베이커리 품목("○○ 기준")**이 항상 표시된다. (기존 "기본 이스트 식빵 기준" 고정 문구를 선택 품목 이름으로 대체)
- **품목 경계 격리**: 진단 결과에는 **선택한 품목의 원인만** 나타난다 — 다른 품목(빵↔제과류 포함)의 원인·증상·판별 질문이 섞이지 않는다.
- **점수 비노출**: 어떤 화면에서도 원시 점수·퍼센트 숫자는 노출되지 않는다 (순위와 정성적 강도 표현만 허용).
- **클라이언트 완결**: 진단 과정에서 외부 네트워크 요청이 발생하지 않는다.
- **식빵 회귀 불변**: v2/v3 확장 후에도 식빵(white-loaf) 경로의 기존 동작(시나리오 1~7)은 바뀌지 않는다.
- **판별 질문 캡의 예외 없는 적용**: 마카롱처럼 실패 모드가 방대하다고 알려진 품목이라도, 판별 질문은 다른 품목과 동일하게 최상위 근접 쌍 1개에 대해 최대 1개만 뜬다 (품목별 예외를 두지 않는다).
