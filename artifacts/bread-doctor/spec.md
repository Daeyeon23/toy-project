# 브레드 닥터 (Bread Doctor) — Spec

## 개요
직접 구운 빵이 실패한 홈베이커가, **만들던 빵 종류를 고르고 → 관찰한 증상을 선택**하면
**가능성 높은 원인을 순위로** 받고 "왜 그런지 + 다음에 고칠 것"을 확인하는 진단 도구다.
상위 원인이 서로 비슷해 가려지지 않을 때는 **판별 질문**으로 좁힌다. 진단 지식은 MVP에서
**기본 이스트 식빵만** 제공되며, 모든 진단은 브라우저 안에서 끝난다(네트워크 없음).

## 범위

### 포함
- **빵 종류 선택 화면** — 여러 빵 계열 카드가 보이되 **식빵만 선택 가능**, 나머지는 "준비 중" 표시
- **기본 이스트 식빵 한 종류** 기준의 증상 기반 진단
- 증상 체크리스트(다중 선택)에서 증상 선택
- **증상 검색·동의어 필터** — 일상어("떡져요")를 입력하면 해당 증상으로 목록이 좁혀짐
- 상위 2~3개 원인을 순위로 제시, 각 원인마다 "원인 / 왜 이 증상이 나오는지 / 다음에 이렇게" 표시
- 상위 원인이 근접할 때만 판별 질문(1~2개)을 띄워 순위를 좁힘
- 증상을 바꿔 다시 진단
- 결과에 "단정적 진단이 아니라 일반적 제빵 상식 기반" 안내 노출

### 제외
- **식빵 외 빵 종류의 진단 지식** — 증상→원인 매핑이 크게 달라 KB가 폭발하고 판별력이 흐려짐. 선택 화면에는 다른 계열이 "준비 중"으로 *보이기만* 하고, 진단은 식빵만 가능. 확장은 식빵 판별력 증명 후.
- **자유 텍스트 해석(LLM/키워드 파싱)** — 결정론(테스트 가능성)을 무너뜨림. 검색·동의어 필터는 *미리 정의된 동의어 사전*으로만 동작하며, 최종 선택은 항상 체크리스트 항목이다.
- **사진 업로드/AI 판독** — 범위 과대. 체크리스트로 가치 증명.
- **계정·로그인·진단 기록 저장** — 데모/포트폴리오 핵심 아님.
- **원시 점수/퍼센트 노출** — 사용자에겐 순위와 정성적 강도만. 스코어는 내부 계산.

## 시나리오

### 0. 빵 종류 선택
- **Given** — 앱 진입 시 빵 종류 선택 화면에 여러 빵 계열 카드(식빵 + "준비 중" 계열 2~3개)가 표시됨
- **When** — "식빵" 카드를 선택함
- **Then** — 식빵의 증상 체크리스트 화면으로 이동함

성공 기준:
- [ ] 선택 화면에 "식빵" 카드와 "준비 중" 배지가 붙은 다른 계열 카드가 함께 표시된다
- [ ] "식빵" 카드 선택 → 증상 체크리스트 화면이 표시된다
- [ ] "준비 중" 계열 카드는 선택해도 증상 화면으로 이동하지 않는다 (비활성)

### 1. 명확한 진단 (판별 질문 없이 바로 결과)
- **Given** — 빵 선택에서 "식빵"을 고른 뒤, 증상 체크리스트가 표시됨
- **When** — "거의 부풀지 않음", "속이 떡짐/질음", "옆구리·표면이 터짐"을 선택하고 "진단하기"를 누름
- **Then** — 판별 질문 없이 결과가 나타나고, 1순위 원인으로 "발효 부족"이 강조되어 표시됨

성공 기준:
- [ ] 위 3개 증상 선택 → "진단하기" → 판별 질문이 나타나지 않고 결과가 표시된다
- [ ] 결과의 1순위 카드 제목이 "발효 부족"이다
- [ ] 결과에 원인 카드가 2~3개 표시된다
- [ ] 각 원인 카드에 원인명, "왜 이 증상이 나오는지" 설명, "다음에 이렇게" 액션 텍스트가 모두 보인다

### 2. 원인-특이적 증상이 순위를 가른다
- **Given** — 진단 화면
- **When** — "부풀다 주저앉음", "기공이 너무 크고 불규칙", "신맛/이상한 냄새"를 선택하고 "진단하기"를 누름
- **Then** — 1순위 원인으로 "과발효"가 표시됨

성공 기준:
- [ ] 위 3개 증상 → 1순위 카드 제목이 "과발효"다
- [ ] "겉은 탔는데 속은 덜 익음" + "겉이 두껍고 딱딱함" 선택 → 1순위 카드 제목이 "오븐 온도가 높음"이다

### 3. 상위 원인이 근접하면 판별 질문이 뜬다
- **Given** — 진단 화면
- **When** — "거의 부풀지 않음" + "속이 떡짐/질음"만 선택하고 "진단하기"를 누름 (이스트 문제 ↔ 발효 부족이 근접)
- **Then** — 결과를 바로 보여주지 않고, 두 원인을 가르는 판별 질문("1차 발효 때 반죽이 부풀긴 했나요?")이 표시됨

성공 기준:
- [ ] 위 2개 증상 → "진단하기" → 판별 질문이 표시되고 최종 순위는 아직 확정되지 않는다
- [ ] 판별 질문에 "예"로 답 → 1순위 카드 제목이 "발효 부족"이다
- [ ] 판별 질문에 "아니오"로 답 → 1순위 카드 제목이 "이스트 문제"다
- [ ] 판별 질문은 최대 2개까지만 이어지고, 그 후에는 결과가 표시된다

### 4. 판별 질문을 건너뛰면 후보를 함께 보여준다
- **Given** — 판별 질문이 표시된 상태 (시나리오 3)
- **When** — "모르겠어요/건너뛰기"를 선택함
- **Then** — 근접했던 두 원인이 순위 결과에 함께 표시됨 (강제로 하나로 좁히지 않음)

성공 기준:
- [ ] 판별 질문에서 "건너뛰기" → 결과에 "이스트 문제"와 "발효 부족" 카드가 모두 표시된다

### 5. 증상 미선택 시 진단 불가
- **Given** — 어떤 증상도 선택되지 않은 초기 화면
- **When** — 화면을 봄
- **Then** — "진단하기" 버튼이 비활성 상태임

성공 기준:
- [ ] 증상 0개 선택 → "진단하기" 버튼이 비활성(클릭 불가)이다
- [ ] 증상을 1개 선택 → "진단하기" 버튼이 활성화된다

### 6. 증상을 바꿔 다시 진단
- **Given** — 시나리오 2의 결과("과발효" 1순위)가 표시된 상태
- **When** — 증상 선택을 "겉은 탔는데 속은 덜 익음" + "겉이 두껍고 딱딱함"으로 바꾸고 다시 "진단하기"를 누름
- **Then** — 이전 결과가 사라지고 1순위가 "오븐 온도가 높음"으로 갱신됨

성공 기준:
- [ ] 결과 표시 후 증상을 바꾸면 이전 결과 카드가 더 이상 보이지 않는다
- [ ] 바뀐 증상으로 다시 진단 → 새 1순위 원인이 표시된다

### 7. 일상어 검색으로 증상 찾기
- **Given** — 증상 체크리스트 화면에 검색 입력란이 표시됨
- **When** — 검색란에 "떡져요"를 입력함
- **Then** — 증상 목록이 "속이 떡짐/질음"으로 좁혀지고, 탭하면 선택된다

성공 기준:
- [ ] "떡져요" 입력 → 목록에 "속이 떡짐/질음"이 나타나고 무관한 증상은 보이지 않는다
- [ ] 필터된 증상을 탭 → 선택 상태가 되고, 검색어를 지우면 전체 목록이 (선택 유지된 채) 복원된다
- [ ] 동의어 사전에 없는 검색어("우주선") 입력 → "일치하는 증상 없음" 안내가 표시된다
- [ ] 검색어 없이도 전체 체크리스트에서 직접 선택할 수 있다 (검색은 보조 수단)

## 불변 규칙

- **면책 고지**: 모든 진단 결과 화면에 "단정적 진단이 아니라 일반적 제빵 상식 기반의 가능성 있는 원인"이라는 안내가 항상 표시된다.
- **범위 고지**: 화면에 "기본 이스트 식빵 기준"임이 항상 표시된다.
- **점수 비노출**: 어떤 화면에서도 원시 점수·퍼센트 숫자는 노출되지 않는다 (순위와 정성적 강도 표현만 허용).
- **클라이언트 완결**: 진단 과정에서 외부 네트워크 요청이 발생하지 않는다.

## 의존성
- 없음. 기존 feature나 외부 시스템에 의존하지 않는다 (정적 진단 지식 기반, 신규 단일 페이지).

## 미결정 항목
- **판별 질문 문안**: 이스트↔발효부족 외에 어떤 근접 쌍에 어떤 질문을 붙일지 — KB 출처 확정(MBT#2, King Arthur 등 인용) 시 함께 정한다.
- **정성적 강도 표현 방식**: "가능성 높음/중간" 라벨 vs 순위 번호만 vs 시각적 강조 — wireframe에서 결정.
- **증상·원인 카피의 최종 문구**: 위 라벨은 초안이며 사용자 용어 검증(Should-Be-True) 후 다듬는다.
- **"준비 중" 계열의 구성**: 어떤 계열을 몇 개 보여줄지(사워도우/크루아상/스콘 중 2~3개) — wireframe에서 결정.
- **동의어 사전의 시드 범위**: 초기 수록 동의어 목록 — 지인 인터뷰(2~3명) 후 확정.
